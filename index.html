This file defines a set of macros and utilities to create a domain-specific language (DSL) style in C++. Below is a reference documentation for all macros included in andromeda.h.

Header Guards

#ifndef ANDROMEDA_H
#define ANDROMEDA_H
#endif

Includes

#include <iostream>
#include <string>
#include "array.h"

Macro Reference

MAIN(code), START(code), BEGIN(code), PLAY(code), LAUNCH(code)

Defines the main entry point with custom naming styles.

cout_start, cout_end, cin_start, cin_end, cout_insrt, cin_insrt

Simplified macros for std::cout and std::cin operations.

input(query, prefix, isEnable)

Handles conditional input with a prefix message.

var, str, bool, int, doub, char, character, variable, integral, string, float, boolean, double

Macros for variable declarations of different types.

script(data), define(name, data), create(name), run(data)

Macros for defining and running functions and code blocks.

pointer, address, data, p, deference, pointerchange

Pointer-related macros to simplify referencing and dereferencing.

index, indexer

Macros for creating switch-case structures.

loop, forplus, forsub, loopforward, loopbackward, forloop

Loop control macros for while-loops and for-loops.

close, use, steal, nest

Namespace manipulation macros.

print, output

Macros for console output with optional prefixes.

iftrue, iffalse, branch

Conditional statement macros.

class, struct, enum, enum2...enum10

Object-oriented macros for class, struct, and enum definitions.

vector, vector2, vector3, vector4

Vector-like integer groupings.

add, sub, multi, div, add2...add4, sub2...sub4, multi2...multi4, div2...div4

Arithmetic helper macros.

on, off, ON, OFF, ENABLE, DISABLE, enabled, disabled

Boolean constants (all evaluate to true).

logic macros: not, nor, and, or, nand, xor, xnor, buffer

Logical operation helpers.

ping, pulse, say, end

Macros for quick output of predefined messages.

Notes

  • Some macros redefine reserved keywords (like int, bool, class), which may cause conflicts.
  • Several macros simplify syntax at the cost of readability and maintainability.
  • Intended for experimentation and DSL-style programming, not for production code.
